# ChampMail — Installation Manual

AI-powered cold-email outreach platform with a conversational CLI, knowledge graph research, and full campaign management.

---

## Quick Install (recommended)

```bash
# Clone both repos side by side
git clone https://github.com/developer00777/Champ_mail.git
git clone https://github.com/developer00777/Graphiti-knowledge-graph.git

# Run the installer (prompts for your OpenRouter key)
cd Champ_mail
bash install.sh
```

That's it. The script handles everything: Docker, Python deps, env files, database, and CLI installation.

To run fully non-interactive (CI / VPS):

```bash
OPENROUTER_API_KEY=sk-or-v1-... bash install.sh
```

To skip the knowledge graph (lightweight install, no AI research):

```bash
bash install.sh --skip-graphiti
```

## Prerequisites

| Requirement | Version | Notes |
|-------------|---------|-------|
| OS | Ubuntu 20.04+ / Debian 11+ / macOS 13+ | Windows: use WSL2 |
| Docker | 24+ | Installed automatically on Linux |
| Docker Compose | v2 plugin | Installed automatically on Linux |
| Python | 3.11+ | For the CLI layer |
| RAM | 4 GB min | 8 GB recommended (Neo4j needs 2 GB) |
| Disk | 10 GB free | Docker images + database |

**Only one API key is required:** an [OpenRouter](https://openrouter.ai/keys) key routes to GPT-4.1, Claude, Gemini — everything.

## What the installer does

```
[1/8]  Install system packages (git, python3, build tools)
[2/8]  Install Docker + Docker Compose
[3/8]  Clone both repositories
[4/8]  Collect your OpenRouter API key, generate all secrets
[5/8]  Write .env files for ChampMail and ChampGraph
[6/8]  Start Neo4j + ChampGraph (knowledge graph)
[7/8]  Build and start ChampMail stack (postgres, redis, backend, worker, frontend)
[8/8]  Install `champmail` CLI globally → accessible from any terminal
```

## First run

Open a new terminal after install (or run `source ~/.bashrc`):

```bash
# Verify all services are up
champmail health check

# Log in with the default admin account
champmail auth login --email admin@champions.dev --password admin123

# Start the conversational AI interface
champmail chat
```

Inside the chat, just type naturally:
- *"let's set up my email"* → guided SMTP/IMAP wizard
- *"I want to email John at john@acme.com"* → full outreach pipeline
- *"show my setup status"* → see what's configured

## Manual setup (after first login)

If you prefer the CLI over chat:

```bash
# Configure your sending domain
champmail setup domain --name outreach.yourcompany.com --provider cloudflare

# Configure SMTP (outbound)
champmail setup smtp \
  --host smtp.yourprovider.com \
  --port 587 \
  --username you@yourcompany.com \
  --password yourpassword \
  --from-email you@yourcompany.com \
  --from-name "Your Name"

# Configure IMAP (reply detection)
champmail setup imap \
  --host imap.yourprovider.com \
  --port 993 \
  --username you@yourcompany.com \
  --password yourpassword

# Set your AI model and key
champmail setup ai --api-key sk-or-v1-... --model openai/gpt-4.1-mini

# Test connections
champmail setup test all

# View everything
champmail setup show
```

## Outreach pipeline

Run the full 6-step pipeline from one command:

```bash
champmail outreach start john@acme.com \
  --first-name John --last-name Doe \
  --title "CTO" --company "Acme Inc" \
  --domain acme.com --industry SaaS \
  --answers-file answers.json
```

Or step by step:

```bash
champmail outreach prospect john@acme.com --first-name John --last-name Doe --title CTO --company Acme
champmail outreach research john@acme.com
champmail outreach questionnaire john@acme.com --answers-file answers.json
champmail outreach prep john@acme.com --save draft.json
champmail outreach send john@acme.com --draft-file draft.json
champmail outreach replies john@acme.com
champmail outreach status john@acme.com
```

`answers.json` format:

```json
{
  "goal":       "book a product demo",
  "pain_point": "manual outreach takes too long",
  "value_prop": "AI-personalised cold email at scale",
  "context":    "they just raised Series A",
  "cta":        "15-min call this week?",
  "tone":       "casual",
  "sender":     "Alex, Head of Growth",
  "from_email": "alex@yourcompany.com"
}
```

## VPS / production deployment

### Firewall

Open these ports on your VPS:

```bash
sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP (if using reverse proxy)
sudo ufw allow 443   # HTTPS
sudo ufw allow 3000  # Frontend (or proxy via nginx)
sudo ufw allow 8000  # Backend API
sudo ufw enable
```

### Environment variables to change for production

Edit `Champ_mail/.env` before starting:

```bash
DEBUG=false
ENVIRONMENT=production
JWT_SECRET_KEY=<strong-random-string>        # change this
EMAIL_ENCRYPTION_KEY=<strong-random-string>  # change this
PUBLIC_API_URL=https://yourdomain.com
FRONTEND_URL=https://yourdomain.com
```

### SMTP for production

Ethereal is a test/catch-all service — emails are captured but not delivered. For real delivery use:

| Provider | SMTP Host | Port |
|----------|-----------|------|
| Gmail (App Password) | smtp.gmail.com | 587 |
| SendGrid | smtp.sendgrid.net | 587 |
| Mailgun | smtp.mailgun.org | 587 |
| Amazon SES | email-smtp.us-east-1.amazonaws.com | 587 |
| Postmark | smtp.postmarkapp.com | 587 |

Configure after install:

```bash
champmail setup smtp --host smtp.sendgrid.net --port 587 \
  --username apikey --password SG.xxxxx \
  --from-email outreach@yourcompany.com
```

## Uninstall

```bash
cd Champ_mail

# Stop and remove all containers + volumes
docker compose down -v

# Stop ChampGraph
cd ../Graphiti-knowledge-graph
docker compose down -v

# Remove CLI symlink
rm -f ~/.local/bin/champmail

# Remove stored config and session
rm -rf ~/.champmail

# Remove the repos (optional)
cd ..
rm -rf Champ_mail Graphiti-knowledge-graph
```

## Troubleshooting

**`champmail: command not found`**
```bash
export PATH="$HOME/.local/bin:$PATH"
# Then add to ~/.bashrc permanently:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
```

**`ModuleNotFoundError: No module named 'cli'`**
```bash
# The champmail script must use readlink to resolve the symlink
# This is already fixed in the current version. Reinstall symlink:
ln -sf "$(pwd)/champmail" ~/.local/bin/champmail
```

**Backend not starting / health check fails**
```bash
docker logs champmail-backend --tail=50
```

**ChampGraph returning 500 errors**
```bash
docker logs graphiti-knowledge-graph-champ-graph-1 --tail=30
# Usually means the OpenRouter key is expired — update it:
champmail setup ai --api-key sk-or-v1-...
# Then restart ChampGraph:
cd ../Graphiti-knowledge-graph && docker compose up -d champ-graph
```

**Neo4j not healthy**
```bash
docker logs graphiti-knowledge-graph-neo4j-1 --tail=30
# Give it 60–90s on first run — it initialises the APOC plugin
```

**SMTP Connection timed out**
This is expected on local machines and sandboxed VPS environments where ports 25/465/587 are blocked by the host firewall. Your email is still recorded in the knowledge graph. Use a cloud provider (Railway, Render, AWS) or a relay service (SendGrid, Mailgun) where SMTP is allowed.

**Docker permission denied**
```bash
sudo usermod -aG docker $USER
newgrp docker   # apply without logging out
```
